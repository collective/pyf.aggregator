# Standards: HTML Heading Structure Fixer

## Code Style

- Follow existing patterns in `rst_to_html.py`
- Use descriptive function names
- Add docstrings to new functions
- Keep functions focused and single-purpose

## Testing Standards

- Use pytest for all tests
- Follow TDD approach (tests first)
- Test edge cases: None, empty string, single H1, multiple H1s
- Test content/attribute preservation
- Use descriptive test names with docstrings
